# Terroir HUB SHOCHU — 制作ルール

## 1. 情報の正確性（最重要）

### 嘘をつかない原則
- **蒸留所が公式に出している情報だけを使う**
- 情報がない項目は「表示しない」。推測で埋めない
- AIが勝手に文章を生成・補完しない

### 引用ルール
- 蒸留所サイト・公式SNSからの文章引用には**必ず出典を明記**する
  ```html
  <div style="font-size:10px;color:var(--text-muted);margin-top:10px;">
    出典：<a href="https://example.com" target="_blank" rel="noopener">蒸留所名 公式サイト</a>
  </div>
  ```
- 杜氏・蔵元の個人名での発言は、**その人物が実際にその発言をしたと確認できる場合のみ**帰属する
- 確認できない場合は蒸留所の一般的な説明として記述する

### データソースの優先順位
1. 蒸留所公式サイト（最優先）
2. ラベル法定表記（アルコール度数・原材料等）
3. 公式コンペティション結果（TWSC / ISC / 鹿児島県本格焼酎鑑評会）
4. 国税庁オープンデータ（蔵名・住所・創業年）
5. 県酒造組合・沖縄県酒造組合連合会の公式情報
6. 蒸留所のSNS公式アカウント

### 使ってはいけないソース
- 個人ブログ・口コミサイトの主観的評価
- AIによる推定・生成テキスト（ファクトチェック不可のもの）
- 出典不明の二次情報

## 2. 銘柄カードの情報設計

### 必須項目（ラベル・公式サイトから取得可能）
- 銘柄名
- 種類（芋焼酎・麦焼酎・米焼酎・黒糖焼酎・泡盛等）
- アルコール度数
- 主原料
- 説明文（公式サイトの記述ベース）

### 条件付き表示項目（データがある場合のみ表示）
- **麹の種類**：黒麹 / 白麹 / 黄麹
  - 蒸留所公開情報がある場合のみ表示
  - 不明な場合は非表示
- **蒸留方法**：常圧蒸留 / 減圧蒸留
  - 公式に記載がある場合のみ
- **熟成**：樽熟成・甕熟成の情報（泡盛の古酒等）
  - 蒸留所が公開している年数のみ
- **受賞歴**：年・コンペ名・賞名
  - 公式発表で確認できるもののみ
  - 受賞歴がない銘柄にはセクション自体を表示しない

### 画像
- 正方形（aspect-ratio: 1/1）のプレースホルダーを標準装備
- 蒸留所から使用許可を得た写真 → `<img class="brand-img">` に差替
- 未許可の蒸留所 → プレースホルダー表示（`brand-img-placeholder`）
- Unsplash等の汎用画像は**蒸留所や銘柄の固有ページには使わない**（誤解を招く）
- ポータルトップの一般的なセクション（地域紹介等）ではUnsplash使用可

## 3. 泡盛の扱い

### 基本方針
- 泡盛は法律上「単式蒸留焼酎」に分類されるが、**文化的に独自の存在**として扱う
- サイト内で焼酎と同格の第一級カテゴリとして表示する
- ナビゲーションでは「焼酎」「泡盛」を並列表示

### 泡盛固有の情報設計
- **古酒（クース）**：3年以上熟成の泡盛。熟成年数を明記
- **仕次ぎ**：伝統的な熟成方法。実施蒸留所は特記
- **原料**：タイ米（インディカ米）＋黒麹菌（全量麹仕込み）
- **度数帯**：25度・30度・43度等。泡盛は度数バリエーションが広い

- **本文**：Noto Sans JP 400, 17px, line-height 1.85
- **UI・数値**：DM Sans
- **最小フォントサイズ**：14px（全世代対応）

### レイアウト
- 銘柄カード：3列グリッド、1px gap、ボーダー区切り
- カードホバー：上部に琥珀色グラデーションライン出現
- border-radius：6〜12px
- ボックスシャドウは控えめに（ボーダー主体）

## 6. SEO・AIO戦略

### 構造化データ（必須）
- 全蒸留所：LocalBusiness + Product schema
- 教科書ページ：FAQPage schema
- 飲み方ガイド：HowTo schema
- パンくずリスト：BreadcrumbList schema
- サイト検索：SearchAction schema

### AIO（AI Overview）対策
- 教科書ページは「問い→答え」形式で構成
- 1段落200文字以内で結論を先に述べる
- 英語版を必ず用意（英語圏のAI Overviewに引用されるため）
